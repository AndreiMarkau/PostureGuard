# PostureGuard 🧘

Десктопное приложение для Windows, которое следит за осанкой через веб-камеру
и издаёт звуковой сигнал при ухудшении позы.

## Требования

- Windows 10/11 (x64)
- Python **3.10 – 3.12** (3.13 не поддерживается MediaPipe)
- Веб-камера с фронтальным обзором (USB или встроенная)
- Камера должна быть **примерно на уровне глаз**, плечи должны быть видны в кадре

## Быстрый старт

```bash
# 1. Распаковать проект
cd posture_fixed

# 2. Установить зависимости и сгенерировать звук
python setup_windows.py

# 3. Запустить приложение
python main.py
```

При первом запуске автоматически откроется окно калибровки.

## Как пользоваться

### 1. Калибровка
При первом запуске откроется окно калибровки:
- Сядьте прямо, смотрите в камеру.
- Убедитесь, что оба плеча видны в кадре.
- Нажмите «Зафиксировать позу» — эталон сохранён.

Повторную калибровку можно запустить кнопкой 🧿 (ScanFace) в нижней панели или через меню трея.

### 2. Главное окно
После калибровки открывается главное окно: превью камеры слева, боковая панель справа.

**Нижняя панель управления:**

| Кнопка | Действие |
|--------|----------|
| 🔊 Звук | Вкл/выкл звуковой сигнал |
| 📷 Камера | Вкл/выкл превью видео (трекинг продолжается) |
| ⚙️ Настройки | Открыть боковую панель на вкладке Настройки |
| ⏸/▶ Пауза/Продолжить | Приостановить или возобновить трекинг |
| 🧿 Калибровка | Открыть окно калибровки |
| 📊 Статистика | Открыть боковую панель на вкладке Статистика |
| ↓ Трей | Свернуть в системный трей |

**Боковая панель** открывается и закрывается кнопкой «›» на границе между камерой и панелью.

### 3. Статистика (вкладка)
- **Текущая серия** — таймер непрерывной хорошей осанки (сбрасывается при нарушении, уходе или паузе)
- **Сегодня** — время сессии и процент хорошей осанки
- **Предупреждения** — счётчик напоминаний о позе и критических предупреждений
- **Недельная статистика** — среднее время в день, качество осанки, дней подряд
- **Таблица рекордов** — открывает отдельное окно с лучшими сериями по уровням строгости

### 4. Настройки (вкладка)

| Параметр | По умолчанию | Описание |
|----------|-------------|----------|
| Уровень строгости: **Мягкий** | 15 сек | Сигнал через 15 сек плохой позы |
| Уровень строгости: **Средний** | 10 сек | Сигнал через 10 сек |
| Уровень строгости: **Жёсткий** | 5 сек | Сигнал через 5 сек |
| Встроенный звук | вкл | Использовать стандартный зуммер |
| Громкость | 80% | Громкость сигнала |
| Свой звук | — | Путь к WAV/MP3/OGG/FLAC-файлу |
| Грейс чай/телефон | 30 сек | Когда рука перекрывает плечи |
| Грейс клавиатура | 10 сек | Взгляд вниз без изменения плеч |
| Away — нет в кадре | 5 сек | Лицо не в кадре → авто-пауза |
| Следить за шеей | вкл | Наклон/опускание головы вперёд |
| Следить за спиной | вкл | Скругление/опускание плеч |
| Язык / Language | — | Русский / English |

### 5. Иконка в трее
Цвет иконки отражает текущее состояние:
- 🟢 Зелёная — осанка в норме
- 🟡 Жёлтая — небольшое отклонение (грейс-период)
- 🔴 Красная — плохая осанка, звуковой сигнал активен
- 🔵 Синяя — пользователь не в кадре (трекинг на паузе)
- ⚫ Серая — трекинг на паузе или остановлен

**Контекстное меню трея** (правая кнопка мыши):
- Открыть окно
- Начать / Пауза / Остановить отслеживание
- Калибровка
- Настройки
- Выход

## Алгоритм детекции

Три метрики вычисляются из координат MediaPipe Pose (model_complexity=2)
и проходят **медианное сглаживание** по скользящему окну 3 секунды (~15 фреймов):

### 1. Наклон головы / «компьютерная шея»
```
H_head = Y_shoulders − Y_ears
Триггер: H_head < H_baseline × threshold  (0.82 / 0.89 / 0.93 по строгости)
```

### 2. Скругление плеч (нормализовано по ширине черепа)
```
Ratio = W_shoulders / W_eyes
Триггер: Ratio < Ratio_baseline × (1 − threshold)  (0.08 / 0.11 / 0.05)
```

### 3. Перекос корпуса в сторону
```
θ = atan2(Δy_shoulders, Δx_shoulders)
Триггер: |θ| > threshold°  (10° / 7° / 5° по строгости)
```

Грейс-периоды позволяют нормально пить чай, смотреть в клавиатуру
или кратковременно менять позу без ложных срабатываний.

## Таблица рекордов

Лучшие серии (≥ 10 секунд) сохраняются автоматически в `records.json`.
Записи разделены по уровню строгости. Открыть таблицу можно кнопкой
«Таблица рекордов» в вкладке Статистика.

## Сборка .exe и установщика

### 1. Собрать PostureGuard.exe

```bat
build_exe.bat
```

Готовый файл появится в `dist\PostureGuard.exe`.

### 2. Собрать Windows-установщик

Требуется [Inno Setup 7](https://jrsoftware.org/isdl.php) (бесплатно).

```bat
installer\build_installer.bat
```

Или вручную:

```bat
iscc installer\PostureGuard_Setup.iss
```

Готовый установщик: `installer\Output\PostureGuard_Setup_1.0.0.exe`

Установщик включает:
- Страницу лицензии (MIT)
- Выбор папки назначения
- Ярлык на рабочем столе (опционально)
- Группу в меню Пуск с деинсталлятором
- Автозапуск при входе в Windows (опционально)
- Корректное обновление поверх предыдущей версии
- Полный деинсталлятор через «Программы и компоненты»

## Ограничения

- Работает только с **фронтальной** камерой (изгиб позвоночника сбоку не определяется)
- Требует хорошего равномерного освещения лица
- При повороте головы более ~30° от камеры точность снижается
- MediaPipe поддерживает Python 3.10–3.12 (не 3.13+)

## Настраиваемые параметры

### 🎯 Строгость (Strictness)

`soft / medium / hard` — определяет **насколько сильно нужно отклониться от эталона**, чтобы система зафиксировала нарушение. Меняет сразу три внутренних порога:

|  | Soft | Medium | Hard |
| --- | --- | --- | --- |
| Голова (drop) | 82% от эталона | 89% | 93% |
| Плечи (ratio) | 8% разница | 11% | 5% |
| Наклон головы | >10° | >7° | >5° |

На практике: **Hard** — сработает при малейшем сутулении; **Soft** — только при выраженном.

---

### ⏱ Grace periods (задержки перед алертом)

**Чай/телефон** (`grace_period_sec`, по умолчанию 30 сек)
Срабатывает когда плечо выходит из кадра (например, потянулся за кружкой или взял телефон). Система ждёт столько секунд, прежде чем засчитать нарушение. Если вернулся раньше — алерт не срабатывает.

**Клавиатура** (`keyboard_grace_sec`, по умолчанию 10 сек)
Срабатывает когда голова опускается вниз, но плечи при этом стабильны — система распознаёт это как взгляд на клавиатуру. Ждёт указанное время перед алертом.

**Away — out of frame** (`away_absence_timeout_sec`, по умолчанию 5 сек)
Сколько секунд человека нет в кадре вообще, прежде чем система решит что ты отошёл и **приостановит** отсчёт нарушений. Если встал и сразу сел — алерт не сработает.

---

### 📋 What to track

**Шея / Neck** (`track_neck`)
Отслеживает вертикальное расстояние от головы до плеч. Реагирует на то, что голова наклоняется вперёд/вниз (типичное «черепашье» положение).

**Спина / Back** (`track_back`)
Отслеживает соотношение ширины плеч в кадре. Когда сутулишься — плечи «схлопываются» и их видимая ширина уменьшается.

Наклон головы в сторону (`tilt`) отслеживается **всегда**, независимо от этих галочек.

---

### 🔊 Звук

**Volume** — громкость алерта (0–100%). Синхронизирована между слайдером в настройках и popup по ховеру.

**Sound file** — кастомный MP3/WAV вместо встроенного сигнала. Если не выбран — используется встроенный.

**Built-in sound (buzzer)** — переключает между встроенным звуком и выбранным файлом.

---

### 🔧 Скрытые параметры (только через config.json)

**`smoothing_window_sec`** (3 сек) — окно медианного сглаживания показаний позы. Чем больше — тем меньше ложных срабатываний от резких движений, но тем медленнее реакция на реальное сутуление.

**`soft/medium/hard_timeout_sec`** (15/10/5 сек) — сколько секунд плохой позы нужно удержать при каждом уровне строгости, прежде чем сработает алерт (это отдельно от grace periods — применяется к обычным нарушениям позы).
